Redesigning the NASA Solar System Treks

A study of the user experience issues and design opportunities of the NASA Solar System Treks website.
The Solar System Trek website aims to introduce public users to space exploration. But so far, they have not tested the usability of the website in public users.
The project is the collaboration between the Interaction Design and Technologies program, Chalmers University and Technology, and Emily Law, the engineer from JPL, NASA. They have been searched for a student to improve the Treks website efficiency. This includes improving the user interface and usability.

Project scope

This project mainly focuses on discovering usability issues on the navigation and layout of the website’s desktop version. The usability problems when using the website in the Virtual Reality and mobile mode did not include in this study. With the limitation of the VR design knowledge, this project didn’t apply any VR design guidelines during development. The advanced feature user interface that requires a permission to access and a Treks portals user interface will not be considered in this project.

Design Process

The double diamond design thinking process is a guideline for this project to follow

Emphathy & Define

The first plan in this project is to understand are there any usability issues on the website, which areas and how critical it is. Before creating the test script, I experienced the website to understand the overall process. And the usability testing was chosen to examine the usability problems.

Usability testing

After the first hands-on experience, the question and tasks were formulated to test on users to discover the problem on the website. The session was conducted on ZOOM and 7 participants were asked to perform the task and think aloud

Usability issues

The problems were categorized based on the level of value and effort of this project’s scope. Some problems were placed in the low value because they are out of scope. This helped me prioritize what I need to focus on in each iteration and prototype

Brainstorm & Sketch

From the feedback, I started to research how space and museum websites organized their content and layout. The interesting mechanism and suitable layouts were chosen and put in on the sketch. It showed that this presentation matches with the info and goal of the trek website or not. Also, I wanted to utilize the old design as much as possible because it is time efficiency for the UI developers team.

Hi - fi prototype

During this process, two iterations of the hi-fi prototype were conducted. In the first iteration, the different navigation, layouts, presentation, fonts, component and color themes were experimented with to find the fit solution. For instance, The current navigation of exploring from the chosen trek portal to the trek portal detail was improved to make a user go to the endpoint faster. For the look and feel, I aim to make a user feel that they are in the universe when exploring the website.

The second iteration mainly worked on refinding the information to show what they can expect from the website, each section and sub-section. For instance, put a short description under each sub-section, showed a sub-section beforehand, visualized a long website's description to a three graphic.

Evaluation

The second usability were conducted for the low-fi prototype with 3 target user via ZOOM. In the usability test, users were assigned the same tasks as a first usability testing to validate an improvement which are about instructions, info layout and navigation. During the session, the participant was asked to think aloud while executing the given tasks.

Two round of usability testing were conducted for hi-fi prototype with 4 participants and UI developers from JPL. The tasks is similar with a previouse one but include the question about look and feel of the website to validate the theme color. More content were put to create a similar environment as the existing website, in order to validate a layout and copy writing. The tasks were given and participants have to complete them while thinking aloud; after each task, participants were asked to rate the difficulty and satifaction rate from 1 to 5, and after the testing, some general questions were brought up to discuss more about the user though of the prototype in detail.

The issues from the first usability testing were solved in each iteration and this is the result from after each test.

Final Design

After three month and many iteration, this is a final desgin. This design shows how the information can be presented to users in many differnets way. How the graphic can capture the users attention and deliver a message more than a text. and how the improved navigation can increase website's efficiency.

Take away

The user research and usability testing are the importance stage of digital product developement. The webiste design might go to the wrong direction. To answer the user experience assumtion from the develpment team, the website should be tested with target users. Foe this website, I found the main works that can improve the usability and working process in the UI team can be seen below.

User research

The website is designed for three different types of users. It is very challenging to design for all in one website because there are different needs for each target. This project is only focused on one target of a user from three main target groups. Also, there are many categories of the target user. In order to design a website that fulfills the user expectation, the team needs to conduct more user researching.

Mobile version

The website aims to support the desktop and mobile version. However, this project did not focus on improving a mobile version and chose to only improve the desktop one because this vision was informed in the middle of the development. For the future work this project should focus on a mobile-first design to create a design that suits both platforms.

Copy writing

One of the issues during the usability testing is a copywriting of the website. The content needs to be writen in an interesting way and easier for the reader to understand and follow.
